# Skill: Session Workflow — Plans & Memory

Управление сессией: планы, память, трекинг прогресса.

## Начало сессии

1. Прочитать `AGENTS.md` — восстановить контекст проекта
2. Прочитать `PLANS.md` — понять предыдущий прогресс
3. Если есть незавершённые задачи — продолжить

## Ведение PLANS.md

`PLANS.md` хранит:
- Активный план сессии с чеклистом
- Контекст и заметки
- Прогресс по задачам

Формат:
```markdown
## Сессия: YYYY-MM-DD — Описание

### Контекст
- ...

### План
1. [ ] Задача 1
2. [ ] Задача 2

### Прогресс
- [x] Сделано
- [ ] В работе
```

## Создание скиллов

Новый скилл создаётся когда:
1. Задача повторяется в разных сессиях
2. Процесс достаточно сложный для пошаговой инструкции
3. Нужно сохранить контекст для будущих сессий

Формат: `skills/<name>.md` с секциями:
- Описание
- Пошаговая инструкция
- Примеры кода
- Чеклист / таблицы

## Конец сессии

1. Обновить `PLANS.md` с прогрессом
2. Обновить `AGENTS.md` если появились новые важные детали
