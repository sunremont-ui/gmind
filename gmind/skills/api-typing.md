# Skill: API Typing — TypeScript Types from Go Structs

## Цель
Автоматическая генерация TypeScript интерфейсов из Go model structs, чтобы типы на фронтенде всегда соответствовали бэкенду.

## Инструмент

`backend/tools/gen-ts-types/main.go` — парсит Go AST из `internal/model/`, генерирует `frontend/src/types/api.ts`.

## Что генерируется

- **Structs** → TypeScript interfaces с JSON-тегами как именами полей
- **Type aliases** (type Foo = string) → `export type Foo = string`
- **omitempty** → optional поля (`?`)
- **Типы**: string, int, float64, bool, time.Time, []T, *T, interface{}, map[K]V

## Поддерживаемые Go-типы → TS

| Go | TS |
|---|---|
| string | string |
| int, float64 | number |
| bool | boolean |
| *bool (omitempty) | boolean? |
| time.Time | string |
| interface{} | unknown |
| []T | T[] |
| map[K]V | Record<K, V> |
| type Foo string | type Foo = string |
| *Struct | Struct (unwrap pointer, omitempty → ?) |

## Исключения (skip)

- `WSMessage` — определяется вручную в `types/index.ts`
- `LayoutNode` — frontend-only renderer type

## Использование

```bash
cd backend && go run tools/gen-ts-types/main.go
```

После генерации:
- `types/api.ts` содержит raw API-типы (source of truth)
- `types/index.ts` ре-экспортирует их и добавляет frontend-специфичные типы (WSMessage, LayoutNode, CursorPosition и т.д.)

## Важно

- **Не редактировать** `api.ts` вручную — изменения перезапишутся
- После изменения Go model structs → запустить генератор
- `api/client.ts` импортирует типы из `../types/api` (API request/response)
- `store/` и компоненты импортируют из `../types` (с ре-экспортом)

## Файлы

- `backend/tools/gen-ts-types/main.go` — генератор
- `frontend/src/types/api.ts` — сгенерированный файл (autogenerated)
- `frontend/src/types/index.ts` — ре-экспорт + frontend-типы
- `backend/internal/model/types.go` — Go structs (источник)
