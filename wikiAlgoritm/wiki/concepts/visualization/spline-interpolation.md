# Сплайн-интерполяция (Spline Interpolation)

## Зачем нужны сплайны

Классическая проблема: есть $n$ измеренных точек (координаты сечения детали, экспериментальные данные, ключевые кадры анимации) — нужно получить гладкую кривую **через** все эти точки.

Наивное решение — полином Лагранжа степени $n-1$ — работает только при малом $n$. При $n > 10$ возникает **эффект Рунге**: дикие осцилляции на краях, кривая далеко отходит от реальных данных между узлами. Использовать нельзя.

Сплайн решает это **кусочно**: строим отдельный полином на каждом интервале, склеиваем их с условиями гладкости. Это даёт точное прохождение через все точки, отсутствие осцилляций, и управляемую гладкость.

**Аналогия**: физический сплайн — гибкая деревянная рейка, прижатая к точкам кнопками. Она принимает форму минимальной упругой энергии изгиба — это и есть естественный кубический сплайн.

## Кубический натуральный сплайн

### Постановка

Дано $n+1$ точек $(x_i, y_i)$. Строим $n$ кубических полиномов $S_i(x)$ на $[x_i, x_{i+1}]$:

$$S_i(x) = a_i + b_i(x-x_i) + c_i(x-x_i)^2 + d_i(x-x_i)^3$$

Условия:
- **Интерполяция**: $S_i(x_i) = y_i$, $S_i(x_{i+1}) = y_{i+1}$ — $2n$ уравнений
- **$C^1$-гладкость**: $S_i'(x_{i+1}) = S_{i+1}'(x_{i+1})$ — $n-1$ уравнений
- **$C^2$-гладкость**: $S_i''(x_{i+1}) = S_{i+1}''(x_{i+1})$ — $n-1$ уравнений
- **Граничные условия** (2 уравнения): естественные ($S''_0(x_0) = S''_{n-1}(x_n) = 0$), зажатые, периодические

Итого: $4n$ неизвестных, $4n$ уравнений → однозначное решение.

### Алгоритм (трёхдиагональная система)

```pseudocode
function cubicSpline(x[], y[]):
    n = len(x) - 1
    h[i] = x[i+1] - x[i]      // длины интервалов

    // Строим трёхдиагональную систему для вторых производных m[i]:
    // Матрица A: [h[i-1], 2*(h[i-1]+h[i]), h[i]] на каждой строке
    rhs[i] = 6 * ((y[i+1]-y[i])/h[i] - (y[i]-y[i-1])/h[i-1])

    m = solveTridiagonal(A, rhs)  // прогонка, O(n)

    // Коэффициенты полинома на i-м интервале:
    for i = 0 to n-1:
        a[i] = y[i]
        b[i] = (y[i+1]-y[i])/h[i] - h[i]*(m[i+1]+2*m[i])/6
        c[i] = m[i] / 2
        d[i] = (m[i+1]-m[i]) / (6*h[i])
```

> 🔢 Сложность: $O(n)$ — трёхдиагональная система решается методом прогонки за линейное время.

### Когда использовать кубический сплайн

**Использовать когда:**
- Нужно точно пройти через все точки (интерполяция, не аппроксимация)
- Данных не слишком много (до ~10 000 точек)
- Одномерная задача (кривая на плоскости или в пространстве, но параметрически)
- Нужна $C^2$-гладкость (непрерывность кривизны)

**Не использовать когда:**
- Данные зашумлены → сплайн пройдёт через шум. Нужна **сглаживающая** аппроксимация (smoothing spline, LSQ)
- Нужно локальное управление → изменение одной точки меняет все коэффициенты
- Многомерные рассеянные данные → RBF (см. [[concepts/numerical/interpolation]])

## B-Сплайны (B-Spline)

### Идея

B-сплайн — обобщение кривых Безье с локальным управлением. Одна управляющая точка влияет только на $k$ соседних сегментов (степень сплайна).

Определяются через **узловой вектор** $T = \{t_0, t_1, \ldots, t_{n+k+1}\}$:

$$C(t) = \sum_{i=0}^{n} P_i N_{i,k}(t)$$

Базисные функции $N_{i,k}(t)$ — рекурсия де Бура:

$$N_{i,1}(t) = \begin{cases} 1 & t_i \leq t < t_{i+1} \\ 0 & \text{иначе} \end{cases}$$

$$N_{i,k}(t) = \frac{t - t_i}{t_{i+k} - t_i} N_{i,k-1}(t) + \frac{t_{i+k+1} - t}{t_{i+k+1} - t_{i+1}} N_{i+1,k-1}(t)$$

```pseudocode
function deBoor(k, t, knots, P):
    // Найти i: t_i <= t < t_{i+1}
    i = findSpan(knots, t)
    d = P[i-k+1 .. i+1].copy()   // k+1 точек

    for r = 1 to k:
        for j = k downto r:
            alpha = (t - knots[j+i-k]) / (knots[j+1+i-r] - knots[j+i-k])
            d[j] = (1-alpha)*d[j-1] + alpha*d[j]

    return d[k]
```

> 🔢 Сложность: $O(k^2)$ на точку (k — степень), $O(1)$ поиск спана для отсортированного узлового вектора

### Типы узловых векторов

| Тип | Вид | Эффект |
|-----|-----|--------|
| Равномерный | $\{0, 1, 2, \ldots, n+k+1\}$ | Кривая не проходит через концы |
| Зажатый (clamped) | $\{0,0,\ldots,0, \ldots, 1,1,\ldots,1\}$ | Кривая начинается и заканчивается на $P_0$, $P_n$ |
| Неравномерный | произвольный | Контроль над параметризацией |

### Когда использовать B-Spline

**Использовать когда:**
- Нужно локальное управление (дизайн автомобильного кузова, интерактивное редактирование)
- Аппроксимация, не интерполяция (кривая приближается к точкам, не проходит через них)
- Степень кривой независима от числа точек
- Нужна $C^{k-2}$-гладкость по всей длине

**Применения:**
- Инструмент свободной формы в CAD (CATIA, SolidWorks, Rhino)
- Описание траектории сварочного робота: гладкая, не требующая прохождения через все промежуточные точки
- Сжатие анимационных кривых: ключевые кадры → B-сплайн, воспроизведение без хранения каждого кадра
- Регрессия данных: аппроксимирующий B-сплайн сглаживает шум измерений

## NURBS (Non-Uniform Rational B-Splines)

### Зачем рациональная форма

B-сплайны — полиномы. Полиномы **не могут точно** описать конические сечения (окружность, эллипс, парабола). NURBS добавляют веса $w_i$:

$$C(t) = \frac{\sum_{i=0}^{n} w_i P_i N_{i,k}(t)}{\sum_{i=0}^{n} w_i N_{i,k}(t)}$$

При $w_i = 1$ → обычный B-сплайн. При разных весах — рациональная кривая.

**Пример**: окружность степени 2 через 9 контрольных точек с весами $\{1, \frac{1}{\sqrt{2}}, 1, \frac{1}{\sqrt{2}}, 1, \ldots\}$.

### Почему NURBS — стандарт CAD

1. **Унификация**: один математический аппарат описывает прямые, дуги, окружности, свободные формы
2. **Точность**: аналитически точные конические сечения, не аппроксимированные
3. **Обменный формат**: STEP (ISO 10303), IGES — международные стандарты, основаны на NURBS
4. **Инвариантность к проективным преобразованиям**: перспективная проекция NURBS = NURBS
5. **Масштабируемость**: от простых профилей шестерён до сложных аэродинамических поверхностей

### Поверхности NURBS

Расширение на 2D:

$$S(u,v) = \frac{\sum_{i=0}^{n}\sum_{j=0}^{m} w_{ij} P_{ij} N_{i,p}(u) M_{j,q}(v)}{\sum_{i=0}^{n}\sum_{j=0}^{m} w_{ij} N_{i,p}(u) M_{j,q}(v)}$$

> 📐 Применение: кузов автомобиля, корпус самолёта, лопатки турбины, корпус редуктора — всё описывается NURBS-поверхностями в CAD-системах.

## Применение по областям

### Машиностроение и CAD
- **Профиль зуба** [[concepts/geometry/involute-gear]]: эвольвента точно описывается рациональной кривой (NURBS)
- **Поверхность редуктора**: корпус, фланцы — NURBS-поверхности в STEP-формате
- **Контроль качества**: CMM (координатно-измерительная машина) измеряет облако точек → аппроксимирующий B-сплайн → сравнение с CAD-моделью
- **Реверс-инжиниринг**: сканирование сломанной детали → сплайн по облаку точек → CAD-модель

### Анимация и кино
- **Motion capture**: данные маркеров (позиции костей) → кубический сплайн → сглаженная анимация без дёрганий
- **Camera path**: путь и ориентация камеры в сцене — B-сплайн по ключевым точкам
- **Morphing**: интерполяция управляющих точек двух NURBS-объектов

### Инженерные расчёты
- **МКЭ на основе изогеометрического анализа (IGA)**: использовать NURBS непосредственно как базис МКЭ — не нужна тесселяция, точность выше. Актуально для задач с тонкими оболочками (кузов, лопатки).
- **Гидродинамика**: NURBS-описание обводов корпуса судна; CFD-расчёт прямо по NURBS без дискретизации

### Медицина
- **Сегментация МРТ/КТ**: граница органа → B-сплайн-контур; 3D-реконструкция органа → NURBS-поверхность
- **Протезирование**: описание сложной поверхности протеза по точкам сканирования

### Геодезия и ГИС
- **Сглаживание маршрутов**: GPS-трек (шумные измерения) → сглаживающий сплайн
- **Интерполяция рельефа**: DEM-точки → NURBS-поверхность → изолинии (горизонтали)

## Сравнение методов

| Метод | Интерполяция | Локальность | Конические сечения | Гладкость | Сложность |
|-------|:-----------:|:-----------:|:-----------------:|:---------:|-----------|
| Кубич. сплайн | Да | Нет | Нет | $C^2$ | $O(n)$ |
| B-Spline | Нет | Да | Нет | $C^{k-2}$ | $O(k^2)$/точка |
| NURBS | Нет | Да | Да | $C^{k-2}$ | $O(k^2)$/точка |
| Bezier кубич. | Нет | — | Нет | $C^\infty$ внутри | $O(n^2)$/точка |
