# Кривые Безье (Bézier Curves)

## Зачем нужны

Кривые Безье решают фундаментальную задачу: как описать плавную произвольную кривую **небольшим набором параметров**, которые художник или инженер может интуитивно двигать мышью. Альтернативы — давать координаты тысяч точек (громоздко) или описывать уравнениями явно (негибко). Безье дают золотую середину: 4 точки → одна плавная кривая.

Безье — это не метод интерполяции (кривая не обязана проходить через управляющие точки, кроме крайних). Это метод **управляемой аппроксимации**: кривая «притягивается» к точкам, но не касается их. Это часто и нужно — поверхность автомобиля не должна иметь перегибов из-за каждой опорной точки.

## Математическое определение

Кривая Безье степени $n$ с управляющими точками $P_0, \ldots, P_n$:

$$B(t) = \sum_{i=0}^{n} \binom{n}{i} (1-t)^{n-i} t^i \, P_i, \quad t \in [0,1]$$

Коэффициенты $B_{i,n}(t) = \binom{n}{i}(1-t)^{n-i}t^i$ — **полиномы Бернштейна**. Они всегда неотрицательны и суммируются в 1 → кривая всегда лежит внутри выпуклой оболочки управляющих точек.

## Алгоритм де Кастельжо (de Casteljau)

Геометрический рекурсивный алгоритм. Вместо вычисления полинома напрямую — линейная интерполяция уровень за уровнем.

```pseudocode
function deCasteljau(P[], t):
    // P — массив управляющих точек
    Q = copy(P)
    n = len(Q) - 1
    for r = 1 to n:
        for i = 0 to n - r:
            Q[i] = (1 - t) * Q[i] + t * Q[i+1]
    return Q[0]  // точка на кривой при параметре t
```

**Почему предпочесть де Кастельжо прямому вычислению полинома:**
- Численно устойчив (нет больших промежуточных чисел)
- Попутно даёт алгоритм разбиения кривой (subdivision) на две подкривые — нужно для рендеринга, пересечений, адаптивной аппроксимации
- Прост в реализации

> 🔢 Сложность: $O(n^2)$ операций на одну точку, $O(n)$ памяти

## Разбиение кривой (Subdivision)

Ключевое свойство: промежуточные точки алгоритма де Кастельжо при параметре $t=0.5$ дают две новые кривые Безье, точно совпадающие с исходной. Это позволяет:
- **Адаптивный рендеринг**: делить кривую рекурсивно, пока сегмент не станет достаточно прямым → аппроксимировать отрезком
- **Нахождение пересечений**: делить обе кривые, проверять пересечение ограничивающих прямоугольников (bounding box) — быстрый rejection

```pseudocode
function subdivide(P[], t=0.5):
    // Возвращает управляющие точки двух подкривых
    left = [], right = []
    Q = copy(P)
    n = len(Q) - 1
    left.append(Q[0])
    for r = 1 to n:
        for i = 0 to n - r:
            Q[i] = (1-t)*Q[i] + t*Q[i+1]
        left.append(Q[0])
        right.prepend(Q[n-r])
    return left, right
```

## Составные кривые и условия гладкости

Для сложных форм соединяют несколько кривых Безье. Степень гладкости в точке соединения:

| Условие | Требование | Визуальный эффект |
|---------|-----------|-------------------|
| $C^0$ | Совпадение конечных точек | Непрерывная кривая, угол |
| $C^1$ | Коллинеарность + равные длины касательных | Гладкое соединение |
| $G^1$ | Только коллинеарность касательных | Визуально гладко, разные скорости |
| $C^2$ | $C^1$ + совпадение вторых производных | Плавное изменение кривизны |

$G^1$ достаточно для визуализации. $C^2$ нужно для инженерных поверхностей (аэродинамика, оптика).

## Степени кривых: когда что использовать

### Линейная (степень 1, 2 точки)
Просто отрезок. Используется внутри алгоритмов (de Casteljau, subdivision).

### Квадратичная (степень 2, 3 точки)
- **Шрифты TrueType** (`.ttf`): каждый контур буквы — набор квадратичных Безье. Выбор исторический: в 1980-х хватало производительности, хранение компактнее, чем кубические.
- **Простые параболические дуги** в UI-анимациях (ease-in/out)
- **Ограничение**: невозможно точно описать окружность (только приближённо)

### Кубическая (степень 3, 4 точки)
- **PostScript / PDF**: весь текст и векторная графика. Стандарт Adobe с 1982 года.
- **SVG** (`C`, `S` команды пути): веб-графика
- **CSS-анимации**: `cubic-bezier(0.25, 0.1, 0.25, 1.0)` — 4 числа задают кривую скорости анимации
- **CAD/CAM**: черновые контуры деталей, траектории инструмента
- **Игры**: траектории снарядов, пути камеры, morph-анимации

### Высокие степени (5+)
Редко используются напрямую — вместо них строят составные кривые. Причина: высокостепенные полиномы осциллируют (эффект Рунге), плохо управляемы.

## Применение по областям

### Компьютерная графика и UI
- Инструмент «Перо» (Pen tool) в Photoshop, Illustrator, Figma — кубические Безье
- Сглаженные переходы между кадрами анимации (motion curves)
- Stroke rendering в векторных редакторах: обводка произвольной ширины вдоль Безье
- Морфинг форм: интерполяция управляющих точек между двумя наборами

### Машиностроение и CAD
- Черновые контуры деталей при 2D-проектировании
- Траектории инструмента в CAM (Computer-Aided Manufacturing): плавный проход фрезы
- Профили лопаток турбин и вентиляторов: $C^2$-гладкость критична для аэродинамики
- Кузовные панели автомобиля: $G^2$-гладкость (совпадение кривизны) устраняет блики-полосы

### Типографика и шрифты
- Описание контуров глифов: каждая буква — набор кривых с точками перегиба
- Хinting: коррекция формы при малых размерах пикселей
- Variable fonts (OpenType): интерполяция управляющих точек между начертаниями (Light ↔ Bold)

### Робототехника
- Планирование траекторий манипулятора: гладкий путь без резких ускорений
- Параметрический путь задаёт скорость ($|B'(t)|$) и ускорение ($|B''(t)|$) в каждой точке

### Геомоделирование (геодезия, ГИС)
- Сглаживание ломаных маршрутов на картах (Google Maps)
- Описание береговых линий, границ природных зон

## Ограничения и когда НЕ использовать

- **Не интерполируют**: если нужно пройти точно через все заданные точки → кубический сплайн (см. [[concepts/visualization/spline-interpolation]])
- **Глобальное управление**: изменение одной точки меняет всю кривую → для локального управления нужны B-сплайны
- **Нельзя точно описать окружность**: только рациональные кривые (NURBS) могут точно — нужно для CAD
- **Высокие степени нестабильны**: для 10+ точек лучше составные кривые или B-сплайны

## Полезные свойства (инварианты)

1. **Аффинная инвариантность**: трансформация кривой = трансформация управляющих точек. Можно поворачивать/масштабировать контрольные точки, результат тот же.
2. **Выпуклая оболочка**: кривая всегда лежит внутри выпуклой оболочки управляющих точек → быстрый rejection при проверке пересечений.
3. **Вариационно-уменьшающая**: кривая не «колышется» больше, чем управляющий многоугольник — нет лишних осцилляций.
4. **Концевые точки**: $B(0) = P_0$, $B(1) = P_n$ — кривая всегда начинается и заканчивается в крайних точках.
5. **Касательные на концах**: $B'(0) = n(P_1 - P_0)$, $B'(1) = n(P_n - P_{n-1})$.
