# Численное интегрирование (Numerical Integration)

Вычисление определённого интеграла $\int_a^b f(x)\,dx$ численными методами.

## Классические формулы

### Метод прямоугольников

$$I \approx h \sum_{i=0}^{n-1} f(x_i), \quad h = (b-a)/n$$

Погрешность: $O(h)$ (левые/правые), $O(h^2)$ (средние точки).

### Формула трапеций

$$I \approx \frac{h}{2}(f(a) + 2\sum_{i=1}^{n-1} f(x_i) + f(b))$$

Погрешность: $O(h^2)$.

### Формула Симпсона

$$I \approx \frac{h}{3}(f(x_0) + 4f(x_1) + 2f(x_2) + 4f(x_3) + \ldots + f(x_n))$$

Погрешность: $O(h^4)$ — точна для полиномов степени $\leq 3$.

## Квадратуры Гаусса (Gauss–Legendre)

Оптимальные узлы и веса для интегрирования на $[-1, 1]$:

$$\int_{-1}^{1} f(t)\,dt \approx \sum_{i=1}^{n} w_i f(t_i)$$

> 🔢 Точна для полиномов степени $\leq 2n-1$ при $n$ точках — оптимальный результат.

```pseudocode
function gaussLegendre(f, a, b, n):
    # t_i, w_i — предвычисленные узлы и веса для [-1, 1]
    t, w = gaussLegendreNodes(n)

    # Преобразование: [-1,1] → [a,b]
    sum = 0
    for i in 0..n-1:
        x = (b-a)/2 * t[i] + (a+b)/2
        sum += w[i] * f(x)

    return (b-a)/2 * sum
```

Стандартные наборы узлов и весов ($n = 2$ до $n = 20$) берутся из таблиц.

## Многомерное интегрирование

### Составные квадратуры (тензорное произведение)

$$\iint f(x,y)\,dx\,dy \approx \sum_{i,j} w_i w_j f(x_i, y_j)$$

> 🔢 Сложность: $O(n^d)$ — «проклятие размерности» при большом $d$.

### Метод Монте-Карло

$$I \approx \frac{V}{N} \sum_{i=1}^{N} f(\mathbf{x}_i), \quad \mathbf{x}_i \sim \text{Uniform}(\Omega)$$

Погрешность: $O(N^{-1/2})$ независимо от размерности — выгоден при $d > 4$.

## Применение в геометрии деталей

> 📐 Применение: вычисление площади поперечного сечения профиля зуба (см. [[concepts/geometry/involute-gear]]) и момента инерции сечения методом квадратур Гаусса по параметрическому описанию контура.

## Адаптивное интегрирование

Алгоритм: применять квадратуру, оценивать погрешность, рекурсивно дробить интервалы с высокой погрешностью.

```pseudocode
function adaptiveSimpsons(f, a, b, tol):
    c = (a + b) / 2
    S = simpsons(f, a, b)
    S2 = simpsons(f, a, c) + simpsons(f, c, b)
    if |S2 - S| < 15 * tol:
        return S2 + (S2 - S) / 15
    return adaptiveSimpsons(f, a, c, tol/2) + adaptiveSimpsons(f, c, b, tol/2)
```
